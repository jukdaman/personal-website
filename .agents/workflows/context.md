# 🌐 JUK's Personal Portfolio - AI Context & Guidelines

본 문서는 JUK(사용자)의 포트폴리오 웹사이트를 개발하기 위해 AI 에이전트가 **필수적으로 숙지하고 따라야 할** 배경 지식, 핵심 용어, 참조 사이트 및 코딩 철학을 밀도 있게 정리한 지침서입니다.

---

## 1. 핵심 용어 정의 (Terminology)
본 프로젝트에서 AI와 사용자는 코스트 및 혼선을 줄이기 위해 아래와 같은 용어를 명확히 구분하여 사용합니다.

- **창 컴포넌트 (Window Component)**: 바탕화면 내에서 띄워지는 모바일/데스크탑 대응 패널. 브라우저 팝업이 아닌, DOM 상에 띄워지는 `div` 기반의 드래그/리사이즈 가능한 객체입니다.
- **1차 창 컴포넌트 (Primary Windows)**: 화면 하단의 내비게이션 바(Nav) 버튼들(ABOUT, WORK, CAREERS, CONTACT)을 눌렀을 때 나타나는 메인 창들입니다. 열려있는 상태에서 다시 Nav 버튼을 누르면 '최소화(닫기)' 처리됩니다.
- **2차 창 컴포넌트 (Secondary Windows)**: 1차 창 컴포넌트 '내부 콘텐츠'를 클릭했을 때 나타나는 서브 창입니다. 1차 창이 닫혀도 2차 창은 별개의 객체(독립적 생명주기)로서 바탕화면에 계속 존재해야 합니다.
- **Singleton Pattern 규칙**: 1차든 2차든 똑같은 내용을 가진 창은 화면에 두 개 이상 중복으로 열릴 수 없습니다 (고유 ID 관리).

## 3. 기술 스택 원칙 및 최적화 철학 (Optimization First)
- **자유로운 프레임워크 선택권 (현재: Vanilla JS)**: 프레임워크 사용에 대한 제약은 없으나, 본 프로젝트의 핵심 로직(수많은 창의 드래그 및 1픽셀 단위 리사이징)은 화면을 매우 높은 빈도로 다시 그리는 작업입니다. Virtual DOM을 거치지 않고 DOM을 직접 컨트롤하는 순수 바닐라 환경이 훨씬 높은 FPS(초당 프레임) 확보에 유리하므로 현재 Vanilla(HTML/CSS/JS) 구조를 채택했습니다.
- **극단적인 최적화 방침**: 저사양 컴퓨터에서도 렉 없이 동작해야 하므로, 움직이는 컴포넌트는 `translate3d`를 통한 하드웨어 가속, `will-change: transform` 명시, 이벤트를 묶어 처리하는 `requestAnimationFrame` 적용이 강제됩니다.

## 4. UI/UX 세부 디테일 수칙
- **윈도우 사이즈**: 초기 크기는 뷰포트에 비례하여 동적으로 결정 (1차 창: `width = vw*0.35`, `height = vh*0.75`, min/max clamp 적용). 블랙 타이틀바 및 오른쪽 끝에 최대화(□), 닫기(X) 버튼 아이콘.
- **최대화/복원 (Maximize/Restore)**: 타이틀바 더블클릭 또는 □ 버튼으로 최대화 토글. 최대화 상태에서는 JS 픽셀이 아닌 CSS `100vw`/`100vh`로 크기를 전환하여 브라우저 리사이징에도 완벽 대응. 최대화 상태에서 타이틀바를 1px이라도 드래그하면 즉시 최대화가 해제되며 마우스 커서 기준으로 복원(Breakout). 최대화 상태에서 가장자리 리사이즈 핸들을 잡으면 화면 크기만한 일반 상태로 강제 복귀 후 리사이즈 연동. 최대화 상태에서 창을 닫아도 다시 열 때는 최대화 이전의 원래 크기/위치값으로 복원됨.
- **창 스폰(Spawn) 위치 계산**: 1차 창 컴포넌트가 열릴 때는 하단 메뉴 버튼의 가로 위치(Index)에 비례하여 화면 좌우에 나란히 배치되고, 순서에 따라 대각선으로 자연스럽게 내려가도록 Y축 오프셋을 줍니다. 
- **상태 기억(Memory)**: 창이 닫혀도, 다시 열면 유저가 리사이즈하거나 옮겨놓았던 크기와 좌표값 그대로 열려야 합니다. 단, 크기가 310x210 이하로 너무 작아졌거나 화면 밖(뷰포트 바깥)에 꽂혀 구출 불가능해진 상태라면 이를 감지하고 기본 Spawning 설정으로 초기화하여 열어주어야 합니다.
- **모바일 대응 (<767px)**: 데스크탑의 '드래그 창' 패러다임이 버려집니다. 대신 기존 데스크탑에서 열어뒀던 여러 창들 중 제일 앞(가장 높은 Z-index)에 켜져 있던 창 객체 하나가 모바일 뷰포트를 꽉 채우는 패널/모달 형태로 변환되어 우선적으로 노출되어야 합니다.
- **유연한 빈 화면 드래그 허용**: 창 이동은 상단 타이틀바뿐만 아니라 내부 흰 본문의 빈 여백 공간을 잡고 드래그해도 지원됩니다. 단, 그 영역 안의 버튼이나 링크 등 인터랙티브 요소를 클릭하거나 텍스트를 복사하는 동작에는 절대 간섭해서는 안 됩니다.
