# Synapse Rendering - 작업 인수인계서 (Handover Document)

새로운 AI 세션 담당자에게: 이 문서는 사용자가 목표로 하는 시각적 퀄리티(Aesthetics)와 지금까지 발생한 문제점, 그리고 다음 세션에서 진행해야 할 명확한 작업 지침을 담고 있습니다.

## 1. 전체 기획 및 목표 (Context & Goal)
*   **프로젝트**: OS 테마 포트폴리오 웹사이트 배경 시각 효과 ('시냅스')
*   **시각적 컨셉 (Shape C - Streamline)**: 허공에서 생성된 수많은 정보 다발들이 보이지 않는 기류(Wind Vector)에 휩쓸려 **S자 유선형 곡선**을 그리며 중앙의 블랙홀로 흡수되는 연출.
*   **핵심 미학 (Core Aesthetics)**:
    1.  **강렬한 결(Stream)의 방향성**: 파티클들이 점선으로 보이지 않고, 거미줄처럼 엮여 있으면서도 중앙 블랙홀을 향해 **진행 방향(안밖축)으로 시원하고 길게 뻗어나가는 스트림**이 되어야 함.
    2.  **부숭부숭함(노이즈) 금지**: 파티클들이 진행 방향을 잃고 양옆(가로축)에 있는 이웃들과 뭉쳐서 지저분한 털뭉치처럼 변하는 현상을 막아야 함.
    3.  **원거리 스케일 불변성**: 카메라 눈앞에 있는 다발과 멀리 있는 다발이 **동일한 밀도와 선명도**를 가져야 함. 멀리 있다고 선이 끊기거나 흐름이 약해 보이면 안 됨.

## 2. 이전 시도와 한계점 (History of Issues)
1.  **순수 3D 물리 거리 연산 (`dist3D < limit`)**:
    *   진행 방향(앞뒤) 간격보다 가로축(양옆) 간격이 물리적으로 더 가까워, 선들이 양옆으로만 거미줄을 치며 흐름이 파괴되고 덩어리지는 치명적 문제 발생.
2.  **가로축 페널티 (`lateralDist`) 및 밀도 정규화 도입**:
    *   양옆으로 뻗는 선에 페널티를 주어 억제하고, 곡선 길이에 비례해 파티클 수를 조절하여 물리적 밀도를 동일하게 맞춤.
    *   **문제점**: 시각적으로 흐름이 어느 정도 개선되었으나, Canvas 렌더링 특성상 원거리 다발의 선이 Sub-pixel 단위로 얇아지며 안티앨리어싱으로 인해 투명해지는 현상을 완전히 잡지 못함. (여전히 원거리 다발이 밋밋함)
3.  **위상수학적 연결(Topological Weaving) 도입 시도와 실수**:
    *   물리적 거리를 무시하고 파티클의 순서(Index) 기반으로 앞서가는 파티클에만 선을 긋는 완벽한 해결책을 고안함.
    *   **[치명적 실수]**: 사용자는 기존 테스트 파일(`synapse_shape_c_streamline.html`)의 구현방식이 가진 한계를 타파하기 위해 **새로운 HTML 파일**에서 코드를 처음부터 다시 짜라고 지시했으나, 반복되는 명령에도 불구하고 AI 에이전트가 계속 기존 코드를 지나치게 참조하고 재사용하려고하려는 경향을 보여줌.
